# logic_tower

logic_tower の入口ページです。
このタワーは、**「論理式を読むのが初めて」**という段階から、
命題論理・述語論理・健全性/完全性・計算可能性・非古典論理までを、
段階的に学ぶための教科書シリーズです。

physics_tower と同様に、単語の定義だけで終わらず、
**直観 → 定義 → 例 → つまずきやすい点 → 確認**の順で理解を積み上げます。

---

## 1. このタワーで到達したい状態
- 命題・述語・証明・モデルの違いを、自分の言葉で説明できる。
- 真理値表や自然演繹を使って、主張の妥当性を確認できる。
- 「証明できること」と「意味的に正しいこと」の関係（健全性・完全性）を説明できる。
- オートマトンや決定可能性と論理学のつながりを理解できる。
- 様相論理・直観主義論理など、古典論理以外の地図を持てる。

## 3. まず最初に覚える用語（最小セット）
- **命題（proposition）**: 真か偽かが定まる文。
- **論理式（formula）**: 記号で書いた命題。
- **解釈（interpretation）**: 記号に意味を与えるルール。
- **モデル（model）**: ある論理式を真にする解釈。
- **証明（proof）**: 推論規則に従って結論へ到達する手続き。
- **妥当（valid）**: どの解釈でも常に真になる性質。

この6語を混同しないだけで、論理学の難しさはかなり減ります。

---

## 4. 典型例（超短い導入）
次の主張を考えます。

- 「雨が降るなら地面は濡れる」
- 「雨が降る」
- したがって「地面は濡れる」

これは命題論理では次の形です。

- `P -> Q`
- `P`
- `Q`

この推論は **modus ponens** と呼ばれ、
論理学で最初に身につける基本規則です。

---

## 5. つまずきやすい点
- 記号の意味を覚える前に、式変形だけを追ってしまう。
- 「証明可能」と「真である（意味的に正しい）」を混同する。
- 量化記号のスコープ（どこまで効いているか）を見落とす。
- 否定の位置（`¬(A ∨ B)` と `¬A ∨ B`）を取り違える。

迷ったら、
1) 日本語に戻す、2) 何を仮定しているかを書く、3) スコープに括弧を付ける、
の3手順で復帰できます。

---

## 6. 学習チェック（自己確認）
- 「妥当」と「証明可能」の違いを1〜2文で説明できる。
- `P -> Q, P ⊢ Q` がなぜ成立するかを真理値表または規則で示せる。
- `∀x P(x)` と `∃x P(x)` を日本語と往復できる。
- 1つの主張について、証明的な見方とモデル的な見方を両方で述べられる。
