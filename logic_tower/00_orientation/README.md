# 00_orientation

`00_orientation` は、logic_tower 全体を読む前に
**「論理学とは何をする学問か」**を整える導入章です。

いきなり記号計算に入るのではなく、
- 何を明確化したいのか
- なぜ自然言語だけでは足りないのか
- 証明とモデルはどう違うのか
を先に共有し、以降の章で迷子にならない土台を作ります。

---

## この章の学習目標
この章を読み終えた時点で、次を目標にします。

- 「論理学の目的」を1〜2文で説明できる。
- 「日常言語」と「形式言語」の違いを具体例で述べられる。
- 「証明的な見方」と「モデル的な見方」の違いを言える。
- 次章（命題論理）で何を学ぶかを把握している。

---

## まず押さえる全体像（3つの問い）
orientation では、次の3問いを中心に進めます。

1. **何を対象にするか？**
   真偽が問える主張（命題）と、その構造。
2. **どうやって正しさを判定するか？**
   推論規則で示す（証明）か、意味づけで確認する（モデル）か。
3. **なぜ記号化するのか？**
   曖昧さを減らし、再現可能な議論にするため。

この3点が明確なら、以降の記号は「暗号」ではなく
「議論を正確にする道具」として読めるようになります。

---

## 直観的な例
次の2文を比べます。

- 「彼は背が高い。」
- 「身長が180cm以上である。」

前者は文脈依存で曖昧ですが、後者は判定基準が明確です。
論理学は、このように**判定可能な形へ整える**方向に進みます。

さらに、
- 「もし雨なら路面は濡れる」
- 「雨である」
- 「ゆえに路面は濡れる」

のような推論が「なぜ正しいか」を、
誰が読んでも同じ手順で確認できる形にするのが論理学の狙いです。

---

## つまずきやすい点
- 記号を覚えること自体が目的だと誤解する。
- 「証明できる」と「実世界で真である」を混同する。
- 具体例を持たないまま抽象語（妥当性・充足など）を追う。

### 対策
- 新しい用語を見たら、必ず1つ日本語例を添える。
- 「これは規則上の主張か、意味上の主張か」を毎回メモする。
- 3行でよいので、自分で推論を書いてチェックする。
